Hugo+GitHub Pagesで自分用ブログを作る

Sat Jul 11, 2020 by 笠原 有真 in  その他  Hugo

環境

  • OS：Windows10 64bit
  • Hugo v0.73.0

導入手順

Hugoのダウンロード

  • ここから最新版のHugoをダウンロードする
  • 今回私はWindows10 64bitだったので，「hugo_0.73.0_Windows-64bit.zip」をダウンロード
  • ダウンロードしたら適当な場所に解凍し，中身の「hugo.exe」をわかりやすいところに置いておく（C:\hugo\とか）

Hugoのパスを通す

  • 「Windowsキー+S」で検索状態にし，「環境変数」と入力してEnter
  • 「環境変数」をクリック
  • 下側の「システム環境変数」の「Path」をダブルクリック
  • 「新規」をクリックして，先ほど「hugo.exe」を置いたパスを指定（私は C:\hugo\）

環境構築

  • 適当なとこに作業用ディレクトリを作成(Documents/workspace的な)
  • PowerShell起動して移動(cd Documents/workspace)
  • hugo new site blog を実行
  • blogディレクトリに移動
  • ここで色々やる

Hugoテーマのダウンロード

  • ここからテーマを探す
    • Hugo Bootstrap v4 Blog
    • Clean White
    • Mainroad
    • あたりがおすすめ（らしい）
  • githubのページからclone用のURLをコピー
  • PowerShellを起動して，./themes に移動したあと，git clone [clone用URL]を実行
  • blog/に戻る
  • すでにあるconfig.tomlを別名にしておいて(mv config.toml config.toml.org)
  • cp .\themes\hugo-theme-bootstrap4-blog\exampleSite\config.toml ./を実行
  • レイアウトとか変えたいときはこのconfig.tomlを変更する

ローカルでのテスト

  • hugo server を実行
  • localhost:1313/　にアクセス
  • ちゃんとページが表示されていればOK
  • タイトルとかはconfig.tomlとにらめっこしながら頑張ろう

新規ポストの作成

  • hugo new posts/sample.md を実行
  • sample.mdファイルをテキストエディタで開いて編集
  • 書き終わったらdraft: false にするのを忘れずに

静的ファイルの生成

  • blogディレクトリに戻る
  • hugo を実行
  • publicディレクトリに静的ファイルが入ってる

GitHubへのアップロード

  • publicディレクトリに移動
  • 以下のコマンドを順番に実行
  • git init
  • git remote add origin [リポジトリのURL]
  • git add -A
  • git commit -m 'コメント'
  • git push origin master
  • なんかエラー出たらググる

GitHubの準備

  • さっきremote addしたリポジトリにアクセス
  • リポジトリの「Setting」の「Options」を開く
  • 「GitHub Pages」欄の「Source」を「master branch」にする
